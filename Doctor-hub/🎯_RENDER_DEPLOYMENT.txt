┌──────────────────────────────────────────────────────────────┐
│  📋 STEP 1: MongoDB Atlas Setup                              │
└──────────────────────────────────────────────────────────────┘

  (Same as before - skip if already done)

  1. Go to: https://cloud.mongodb.com/
  2. Sign up (FREE account)
  3. Create Cluster:
     - Click "Build a Database"
     - Select M0 (FREE tier)
     - Region: Singapore/closest
     - Cluster Name: DoctorHubCluster
  
  4. Create User:
     - Username: doctorhub_user
     - Password: (strong password - SAVE IT!)
  
  5. Network Access:
     - Click "Network Access"
     - Add IP: 0.0.0.0/0 (Allow from anywhere)
  
  6. Get Connection String:
     - Click "Connect"
     - "Connect your application"
     - Copy string:
       mongodb+srv://doctorhub_user:PASSWORD@cluster.mongodb.net/doctor-hub
     - Replace PASSWORD with your actual password
     - SAVE THIS STRING!

┌──────────────────────────────────────────────────────────────┐
│  📋 STEP 2: Deploy Backend                                   │
└──────────────────────────────────────────────────────────────┘

  1. Go to: https://render.com/
  
  2. Sign Up:
     ✓ Click "Get Started for Free"
     ✓ Sign up with GitHub (RECOMMENDED)
     ✓ Authorize Render to access your repos
  
  3. Create Backend Service:
     ✓ Click "New +" button (top right)
     ✓ Select "Web Service"
     ✓ Find your repository (doctor-hub)
     ✓ Click "Connect"
  
  4. Configure Backend:
     ┌────────────────────────────────────┐
     │ Name: doctor-hub-backend           │
     │ Root Directory: backend            │
     │ Environment: Node                  │
     │ Region: Singapore (or closest)     │
     │ Branch: main                       │
     │ Build Command: npm install         │
     │ Start Command: npm start           │
     │ Instance Type: Free                │
     └────────────────────────────────────┘
  
  5. Add Environment Variables:
     Click "Advanced" → "Add Environment Variable"
     
     ┌──────────────────────────────────────────────┐
     │ NODE_ENV = production                        │
     │ PORT = 10000                                 │
     │ MONGODB_URI = [Your MongoDB string]          │
     │ CLIENT_URL = https://temp.com (update later) │
     │ JWT_ACCESS_SECRET = doctor_hub_access_...    │
     │ JWT_REFRESH_SECRET = doctor_hub_refresh_...  │
     │ JWT_ACCESS_EXPIRES = 15m                     │
     │ JWT_REFRESH_EXPIRES = 7d                     │
     └──────────────────────────────────────────────┘
  
  6. Create Service:
     ✓ Scroll down
     ✓ Click "Create Web Service"
     ✓ Wait 5-10 minutes (first deploy is slow)
  
  7. Get Backend URL:
     ✓ After deployment completes
     ✓ Copy URL (e.g., https://doctor-hub-backend.onrender.com)
     ✓ SAVE THIS URL!
  
  8. Test Backend:
     ✓ Open in browser: YOUR_BACKEND_URL/api/health
     ✓ Should show: {"success": true, ...}
     ✓ If YES → Backend is LIVE! ✅

┌──────────────────────────────────────────────────────────────┐
│  📋 STEP 3: Deploy Frontend                                  │
└──────────────────────────────────────────────────────────────┘

  1. Create Frontend Service:
     ✓ Click "New +" again
     ✓ Select "Static Site"
     ✓ Connect SAME repository
  
  2. Configure Frontend:
     ┌─────────────────────────────────────────────┐
     │ Name: doctor-hub-frontend                   │
     │ Root Directory: frontend                    │
     │ Branch: main                                │
     │ Build Command: npm install && npm run build │
     │ Publish Directory: frontend/dist            │
     └─────────────────────────────────────────────┘
  
  3. Add Environment Variable:
     Click "Advanced" → "Add Environment Variable"
     
     ┌──────────────────────────────────────────────┐
     │ VITE_API_URL = YOUR_BACKEND_URL/api          │
     │                                              │
     │ Example:                                     │
     │ https://doctor-hub-backend.onrender.com/api  │
     └──────────────────────────────────────────────┘
  
  4. Create Site:
     ✓ Click "Create Static Site"
     ✓ Wait 3-5 minutes
  
  5. Get Frontend URL:
     ✓ After deployment completes
     ✓ Copy URL (e.g., https://doctor-hub-frontend.onrender.com)
     ✓ SAVE THIS URL!

┌──────────────────────────────────────────────────────────────┐
│  📋 STEP 4: Update Backend CLIENT_URL                        │
└──────────────────────────────────────────────────────────────┘

  1. Go to Backend Service:
     ✓ Render Dashboard → doctor-hub-backend
  
  2. Update Environment Variable:
     ✓ Click "Environment" tab (left sidebar)
     ✓ Find "CLIENT_URL"
     ✓ Click "Edit"
     ✓ Replace with YOUR FRONTEND URL
     ✓ Example: https://doctor-hub-frontend.onrender.com
     ✓ NO TRAILING SLASH!
     ✓ Click "Save Changes"
  
  3. Wait for Auto-Redeploy:
     ✓ Service will automatically redeploy
     ✓ Wait 2-3 minutes
     ✓ Check "Events" tab for status

┌──────────────────────────────────────────────────────────────┐
│  ❌ TROUBLESHOOTING                                          │
└──────────────────────────────────────────────────────────────┘

  Problem: Build Failed
  ─────────────────────
  Solution:
    • Check Render logs (Dashboard → Logs)
    • Verify package.json exists
    • Check Node version (Render uses latest LTS)
    • Root directory correct hai?

  Problem: Cannot Connect to Database
  ───────────────────────────────────
  Solution:
    • MONGODB_URI sahi hai?
    • MongoDB Atlas Network Access 0.0.0.0/0 allowed?
    • Database user password correct hai?
    • Connection string mein password special characters?
      (URL encode karo: @ = %40, # = %23)

  Problem: CORS Error
  ───────────────────
  Solution:
    • CLIENT_URL exactly frontend URL ke equal hai?
    • No trailing slash at end
    • Backend redeploy hua after CLIENT_URL update?

  Problem: 502 Bad Gateway
  ────────────────────────
  Solution:
    • Wait a few minutes (first request is slow on free tier)
    • Check backend logs
    • MongoDB connection working hai?

  Problem: Static Site Not Loading
  ────────────────────────────────
  Solution:
    • Build command sahi hai?
    • Publish directory "frontend/dist" hai?
    • VITE_API_URL sahi set hai?
    • Check browser console for errors
