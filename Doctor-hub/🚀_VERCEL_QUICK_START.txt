┌──────────────────────────────────────────────────────────────┐
│  📋 DEPLOYMENT STEPS / تفصیلی قدم                           │
└──────────────────────────────────────────────────────────────┘

  STEP 1: MongoDB Atlas Setup (Required!)
  ────────────────────────────────────────
  1. Go to: https://cloud.mongodb.com/
  2. Create FREE account
  3. Create database cluster (M0 Free tier)
  4. Create database user
  5. Allow network access (0.0.0.0/0)
  6. Get connection string
  
  📚 Detailed Guide: VERCEL_DEPLOYMENT_URDU.md (Step 1)

  STEP 2: Push Code to GitHub (If not done)
  ────────────────────────────────────────
  git init
  git add .
  git commit -m "Ready for Vercel deployment"
  git remote add origin YOUR_GITHUB_REPO_URL
  git push -u origin main

  STEP 3: Deploy Backend on Vercel
  ────────────────────────────────────────
  1. Go to: https://vercel.com/dashboard
  2. Import your GitHub repository
  3. Configure:
     - Root Directory: backend
     - Framework: Other
  4. Add environment variables:
     - MONGODB_URI (from Step 1)
     - CLIENT_URL (will update later)
     - JWT_ACCESS_SECRET
     - JWT_REFRESH_SECRET
     - Other variables from backend/.env
  5. Deploy!
  6. Copy backend URL
  
  📚 Detailed Guide: VERCEL_DEPLOYMENT_URDU.md (Step 2)

  STEP 4: Deploy Frontend on Vercel
  ────────────────────────────────────────
  1. Import SAME repository again
  2. Configure:
     - Root Directory: frontend
     - Framework: Vite
     - Build Command: npm run build
     - Output: dist
  3. Add environment variable:
     - VITE_API_URL = YOUR_BACKEND_URL/api
  4. Deploy!
  5. Copy frontend URL
  
  📚 Detailed Guide: VERCEL_DEPLOYMENT_URDU.md (Step 3)

  STEP 5: Update Backend CLIENT_URL
  ────────────────────────────────────────
  1. Go to backend project settings
  2. Update CLIENT_URL to your frontend URL
  3. Redeploy backend
  
  📚 Detailed Guide: VERCEL_DEPLOYMENT_URDU.md (Step 4)

  STEP 6: Test Your Deployment!
  ────────────────────────────────────────
  1. Open backend URL: /api/health
     Should return: {"success": true, ...}
  
  2. Open frontend URL
     Should show login page
  
  3. Try login (if demo data seeded)
  
  📚 Detailed Guide: VERCEL_DEPLOYMENT_URDU.md (Step 5)

┌──────────────────────────────────────────────────────────────┐
│  ❌ COMMON ISSUES / عام مسائل                               │
└──────────────────────────────────────────────────────────────┘

  Issue: "Cannot connect to database"
  Solution: 
    - MongoDB Atlas connection string verify karo
    - Network Access "Allow from Anywhere" hai check karo
    - Database user credentials correct hain?

  Issue: "CORS Error"
  Solution:
    - Backend CLIENT_URL mein exact frontend URL hona chahiye
    - No trailing slash at the end
    - Backend redeploy karo after updating

  Issue: "502 Bad Gateway"
  Solution:
    - Vercel logs check karo (Dashboard → Deployments)
    - MongoDB connection working hai verify karo
    - All environment variables set hain?

  Issue: "Module not found"
  Solution:
    - package.json mein dependency hai check karo
    - "type": "module" hai verify karo
    - Import paths correct hain (.js extension)?
