    • Go to: https://cloud.mongodb.com/
    • Create FREE database cluster
    • Create database user
    • Get connection string
    📖 Guide: VERCEL_DEPLOYMENT_URDU.md (Step 1)

2️⃣  PUSH TO GITHUB (if not done)
    git init
    git add .
    git commit -m "Ready for deployment"
    git push

3️⃣  DEPLOY BACKEND
    • Vercel Dashboard → Import Project
    • Root Directory: backend
    • Add environment variables
    • Deploy!
    📖 Guide: VERCEL_DEPLOYMENT_URDU.md (Step 2)

4️⃣  DEPLOY FRONTEND
    • Import same repo again
    • Root Directory: frontend
    • Add VITE_API_URL
    • Deploy!
    📖 Guide: VERCEL_DEPLOYMENT_URDU.md (Step 3)

5️⃣  UPDATE BACKEND
    • Update CLIENT_URL with frontend URL
    • Redeploy backend
    📖 Guide: VERCEL_DEPLOYMENT_URDU.md (Step 4)

6️⃣  TEST & ENJOY!
    • Test backend: /api/health
    • Test frontend: login page
    • Your app is LIVE! 🎉

 🎓 KEY POINTS TO REMEMBER

1. MongoDB Atlas REQUIRED
   - Local MongoDB won't work on Vercel
   - Must use cloud database (Atlas)
   - FREE tier available

2. Deploy Backend and Frontend SEPARATELY
   - Backend: One Vercel project
   - Frontend: Another Vercel project
   - They communicate via URLs

3. Environment Variables MUST be set
   - In Vercel Dashboard → Settings
   - NOT in code
   - Different for backend and frontend

4. CLIENT_URL must be EXACT
   - No trailing slash
   - Must match frontend URL exactly
   - Update after frontend deployment

5. GitHub Push Triggers Auto-Deploy
   - After initial setup
   - Every push deploys automatically
   - Check Vercel dashboard for status
